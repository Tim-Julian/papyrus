Releng

Context

Goal is to be able to release your project generated with Papyrus Components

The following procedure works if and only if all plugins have the same version, ie you will upgrade same in batch mode.

Prerequisites

Ensure your Hudson jobs are green https://hudson.eclipse.org/papyrus/view/COMPONENTS/job/.

Ensure that no critical bugs are opened https://bugs.eclipse.org/bugs/buglist.cgi?bug_status=__open__&component=Components&product=Papyrus&query_format=specific

Have a look to the nightly documentation in order to fetch the needed repository: https://hudson.eclipse.org/papyrus/view/Components/job/papyrus-components-website/ws/site-staging/index.html

Actions

  • Create a bugzilla ticket [Components][releng] Release of version X.Y.Z
  • Fetch your repository
  • Modify the change log located in the org.eclipse.papyrus.components.parent/src/changes/changes.xml
  • Change from .qualifier -SNAPSHOT to a stable version, see CommandA
  • Change your target to match a specific release version, see commandB
  • Merge these modifications
  • Run your associated job Release Components: papyrus-component
  • It should deploy the new version into the nexus repository and execute downstream the website job Components: papyrus-components-deploy-website. The downstream jobs can be seen at the end of the build's console report.
  • Deploy the web site by copying the staging directory in the papyrus web git repository. To do this, first clone the Papyrus Web git repository and unzip in the components/components/ directory the unzipped built site from papyrus-components-deploy-website
  • Modify the index.html to reflect the new release version by adding a new item to the dropdown menu and updating the referenced page address. This file can be found in the Papyrus Web git repository at the root of the components/components/ folder
  • Increase the different pom.xml, manifest.mf artifact version by adding a +(0.0.1)-SNAPSHOT.qualifier suffix in parent and archetype. See CommandA
  • Restore the 0.0.0 in all your target files
  • Merge these modifications
  • Close the ticket release.

CommandA example of command to upgrade pom.xml, plugin.xml, manifest.mf, category.xml, feature.xml etc..., check if it runs on all plugins depending on your profile configuration 

mvn org.eclipse.tycho:tycho-versions-plugin:set-version -DnewVersion=1.2.2.qualifier -DupdateVersionRangeMatchingBounds=true -Pproduct

CommandB example of command to upgrade your targets that will switch from a 0.0.0 to a specific version

mvn org.eclipse.tycho.extras:tycho-version-bump-plugin:update-target -Dtarget=/home/flefevre/gitNeon/org.eclipse.papyrus-marte/targetplatform/org.eclipse.papyrus.marte.targetplatform.neon/org.eclipse.papyrus.marte.targetplatform.neon.target

Post-Actions

You could inform Papyrus Components project leader that you have improve the project in order they switch to the new pom version.
